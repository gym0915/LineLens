把它和 ChatGPT 并排打开，会发现一件有趣的事：两家公司，把最重要的产品入口——对话输入框——做成了几乎一模一样的样子。一个文本框、一个加号、一个语音按钮，仅此而已。
当一个产品形态被反复打磨、被海量用户验证之后，最优解往往会向同一个点靠拢。
这个设计是怎么样的？
一级输入栏，只保留了三个元素：
  1. 加号（+） —— + 什么并没有说明，添加附件？
  2. 文本输入框 —— 主要交互入口
  3. 音频按钮 —— 语音模式
AI 对话场景的输入方式核心交互场景其实就是：Text Message - 打字，加附件，照片或者其他。
除此之外，可选的交互还有
  • 模型选择
  • Thinking
  • Web search
  • 语调
  • ...
也许很多人还记得，ChatGPT在一段时间内，不断的在改造这个 Input Bar，直到有一天，上面这个样式出现了，一直到现在，无论模型、新产品能力如何更新，这个 Input Bar 都保持了这样子。这就是设计的最佳实践了。

为什么这么设计？

模型选择降低权重
两款产品都没有把"选择模型"这件事放进输入栏。Gemini 把模型切换放在顶栏标题下拉里，ChatGPT 同样。它们的潜台词是一致的：
我们不希望用户在每次提问前都先思考一遍"我现在应该用 GPT-4o 还是 o1？"
从交互设计的角度 —— 决策项越多，决策时间越长。模型本质上是一个工程实现的细节。把它前置到输入栏，相当于强迫用户每次都做一次"我配不配用最好的模型"的自我审查，这种心智负担是反人类的。
把模型入口放远一点、让它"不那么容易点到"，看似是反直觉的设计，其实是一种对默认值的信任——产品自信地告诉用户：你不用管这个，我们替你选好了。

「+」 号图标，就是这里最精彩的部分，他是一个 + 号，不是附件，不是图片。
具体的，这个容器实现了：
  1. Sheet 的层级感天然适配"主任务—辅助任务"的关系。它从底部升起，不抢占输入区域的视觉焦点，用完即收，符合渐进式披露（Progressive Disclosure）的原则。
  2. Sheet 把"添加内容"与"调用能力"放在了同一个空间。打开 ChatGPT 的加号，你会看到相册、相机这类"加东西进来"的入口，紧接着是 Create image、Deep research、Web search 这类"调用能力出去"的入口。它们在用户心智里同属"我现在想让对话变得更丰富一点"，所以应该在同一个容器里出现。
  3. Sheet 提供了一种"温柔的发现"。高级功能没有被丢进设置页面，也没有挤在一级 Toolbar 上吓退新用户。它们躺在加号背后，等着那些已经熟悉基础对话、开始有进阶需求的用户去主动探索。这就是降低发现成本最优雅的方式——不解释、不引导，只是恰好出现在用户已经准备好的那一刻。
