# LineLens

长文阅读最大的问题，往往不是文章太长，而是注意力很难稳定停在当前这一句。

你可能也有过这样的体验：打开一篇很想认真读完的技术文章、行业分析或深度评论，刚读了几段，视线就被旁边的按钮、推荐流、嵌入卡片、评论入口和复杂排版拉走。读着读着，突然发现自己不知道刚才读到哪里；再往回找，又打断了原本的理解节奏。

LineLens 想解决的就是这个问题。

它把原本散落在媒体平台里的长文，整理成一个安静、干净、可持续推进的阅读空间。用户不需要和页面噪音对抗，也不需要靠意志力记住当前位置。LineLens 会把文章拆成适合阅读的语义单元，让你每次只专注眼前这一小段，同时保留上下文，读下去不会散。

## 这是谁需要的产品

LineLens 面向的是那些真的想把一篇长文读进去的人。

如果你经常阅读技术长文、AI/产品/投资分析、创作者 Newsletter、长篇观点文章，LineLens 会很适合你。你不缺内容来源，真正缺的是一个能让你稳定读完内容的环境。

它也适合那些容易跳读、漏读、读到一半忘记上下文的人。很多时候问题不在理解能力，而在阅读界面不断制造干扰：正文之外的信息太多，段落节奏太乱，当前视线没有可靠锚点。

LineLens 的目标不是帮你“更快刷完信息”，而是帮你把值得读的内容真正读完。

## 用户正在遇到什么障碍

今天很多长文并不是在阅读器里被消费，而是嵌在社交平台、订阅平台和媒体页面里。页面本身承担了太多目标：互动、转发、推荐、订阅、跳转、广告、嵌入内容、媒体播放。

这些元素对平台有价值，但对一个正在认真阅读的人来说，都是额外负担。

典型的阅读障碍包括：

- 正文周围有太多按钮和推荐内容，注意力随时被带走。
- 段落很长，读完一句后很难自然接住下一句。
- 图片、视频、嵌入推文、代码块和表格混在正文里，阅读节奏被反复打断。
- 原网页排版不稳定，字体、间距、颜色和媒体尺寸不一定服务于专注阅读。
- 多次打开同一篇文章时，很难回到上次真正读到的位置。
- 阅读时明明内容有价值，但界面不断让人产生“算了，等下再读”的感觉。

LineLens 不是简单地把网页变白、把广告藏掉。它关注的是一个更细的体验：当用户读到某一句、某一段、某个图片或代码块时，系统能不能帮助他稳稳地停在那里。

## LineLens 怎么解决

LineLens 的核心体验是“聚焦式阅读”。

当你打开一篇支持的长文，点击浏览器里的 LineLens 图标，文章会进入独立 Reader 页面。这个页面会去掉平台噪音，只保留真正和阅读有关的内容：标题、正文、列表、引用、图片、代码、表格、视频和必要的嵌入内容。

更重要的是，LineLens 不只是把文章重新排版。它会把正文拆成适合阅读节奏的语义单元。当前正在阅读的部分会被清晰高亮，其余内容会自然退到背景里，但仍然能被感知。这样你知道自己正在读哪里，也知道上下文在哪里。

阅读可以像翻页一样向前推进：

- 使用左右方向键切换上一个或下一个阅读单元。
- 点击任意段落、图片或内容块，可以直接把阅读焦点移动过去。
- 重新打开同一篇文章时，可以从上次的位置继续读。
- 图片加载、窗口尺寸变化或字体变化后，当前高亮仍会跟随内容位置。

这种体验的价值不是炫技，而是降低阅读时的心智负担。你不需要一直提醒自己“别跳走”，界面会帮你把注意力留在当前内容上。

## 今天它已经能做到什么

LineLens 现在已经可以完成从“发现一篇值得读的文章”到“进入专注阅读状态”的完整过程：

- 遇到支持的长文页面时，LineLens 会提示这篇内容可以进入阅读模式。
- 点击扩展图标后，文章会在独立阅读页中打开。
- 正文会被重新整理成更干净的阅读版面，弱化平台干扰。
- 当前正在读的语义单元会被高亮，周围内容保留为上下文。
- 可以用键盘一段一段推进，也可以点击任意位置重新定位阅读焦点。
- 离开后再回来，可以接着上次停下的位置读。
- 标题、段落、列表、引用、图片、图集、代码、表格、链接、视频和嵌入内容都能进入同一个阅读节奏。
- 阅读页支持系统主题、本地字体、深浅色体验，以及非当前内容的自然弱化。
- X / Twitter 长文里的图片、GIF、视频、代码块、表格、媒体说明和嵌入推文可以被保留下来。
- Substack 文章里的正文、标题、列表、引用、图片、代码、表格和 Twitter 嵌入内容可以被保留下来。

## 当前支持的平台

- X / Twitter Articles：`x.com/{user}/article/{id}`、`twitter.com/{user}/article/{id}`
- Substack：`substack.com/inbox/post/*`、`substack.com/home/post/*`、`substack.com/p/*`、`*.substack.com/p/*`

## 典型使用场景

当你在 X / Twitter 上看到一篇值得认真读的长文，不需要在原页面里和时间线、互动按钮、媒体卡片一起抢注意力。打开 LineLens 后，它会变成一个更适合阅读的页面，你可以一段一段往下推进。

当你阅读 Substack 文章时，LineLens 会把正文和图片、引用、列表、嵌入内容放进统一的阅读节奏里。你看到的不再是平台页面，而是一篇可以被稳定读完的文章。

当你只能分几次读完一篇内容时，LineLens 会保存阅读位置。下次回来，不需要凭记忆找段落，可以继续从上次停下的地方读。

## 安装部署

克隆项目：

```bash
git clone git@github.com:gym0915/LineLens.git
cd LineLens
```

安装依赖：

```bash
npm install
```

构建扩展：

```bash
npm run build
```

构建完成后，浏览器可加载的扩展目录是：

```text
LineLens/dist
```

在 Chrome 或 Edge 中打开扩展管理页，开启开发者模式，选择“加载已解压的扩展程序”，然后选择上面的 `dist` 目录。

更新代码后重新执行：

```bash
npm run build
```

然后在浏览器扩展管理页中重新加载 LineLens。
